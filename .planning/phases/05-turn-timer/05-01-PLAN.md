    Extend the `GameMessage` discriminated union with three new variants:
    ```
    | { type: 'timer-config'; mode: TimerMode }
    | { type: 'timer-sync'; remaining: number }
    | { type: 'timer-expired' }
    ```

    - `timer-config`: Sent by host to guest on game-start to communicate the timer setting. `mode` is 30, 60, or 0 (unlimited).
    - `timer-sync`: Sent by host to guest periodically (~5s) and on turn changes. `remaining` is whole seconds left.
    - `timer-expired`: Sent by host to guest when the timer hits 0 and the turn is being forfeited.

    The `game-start` message already sends the initial snapshot. The `timer-config` message is sent immediately after `game-start` (only if mode !== 0).
