    let timerRemaining = $state(0);
    let timerRunning = $state(false);
    let timerExpired = $state(false);
    let shaking = $state(false);
    const timerWarning = $derived(timerRunning && timerRemaining <= TIMER_WARNING_THRESHOLD && timerRemaining > 0);

    **Host timer logic:**
    - Store `timerStartedAt: number` (Date.now() when timer started) and `timerDuration: number` (seconds) as local variables (not $state -- not reactive).
    - `startTimer()`: Sets `timerStartedAt = Date.now()`, `timerDuration = timerMode`, `timerRunning = true`, `timerExpired = false`. Creates a `setInterval` at 200ms (per research code example for smooth self-correcting updates). Each tick: compute `remaining = Math.max(0, Math.ceil((timerStartedAt + timerDuration * 1000 - Date.now()) / 1000))`. Update `timerRemaining = remaining`. If `remaining <= 0`: clear interval, set `timerExpired = true`, `timerRunning = false`, trigger forfeit sequence.
    - `stopTimer()`: Clears interval, sets `timerRunning = false`.
    - `resetTimer()`: Calls `stopTimer()`, then `startTimer()` if timerActive.
    - Store the interval ID to clear on stopTimer/destroy.

    **Timer sync (host side):**
    - Add a sync counter. Every 25 ticks (25 * 200ms = 5 seconds), send `{ type: 'timer-sync', remaining: timerRemaining }` to guest.
    - On turn change (after `applyMove` switches `currentPlayer`), call `resetTimer()` and send `{ type: 'timer-sync', remaining: timerMode }`.

    **Timer forfeit sequence (host side, per D-08, D-09, D-10):**
    - When timer expires: set `shaking = true`. After a 1000ms `setTimeout`: call `forfeitTurn(snapshot)` to advance the turn, send `{ type: 'timer-expired' }` to guest, send `{ type: 'state-update', snapshot }`, set `shaking = false`, call `resetTimer()` for the new player's turn.
    - Guard: if `snapshot.status !== 'playing'`, skip forfeit (Pitfall 2).

    **Host onConnect handler update:**
    - After sending `game-start`, if `timerActive`, send `{ type: 'timer-config', mode: timerMode }` and call `startTimer()`.

    **Host onData handler for move-request:**
    - After a valid move is applied, check if the turn changed (new `snapshot.currentPlayer !== previousPlayer`). If so, call `resetTimer()` and send timer-sync with full duration.

    **Host rematch handler:**
    - Clear timer. After rematch snapshot is created and sent, if timerActive, send `{ type: 'timer-config', mode: timerMode }` and call `startTimer()`.

    **Guest timer logic:**
    - When `timer-config` message received: store the mode locally, set `timerRemaining = mode`, `timerRunning = true`. Start a local display interval (200ms) that decrements based on elapsed time from a local start timestamp. This is purely for smooth display -- host is authoritative.
    - When `timer-sync` message received: update `timerRemaining = msg.remaining`, reset the local display timer start point.
    - When `timer-expired` message received: set `timerExpired = true`, `shaking = true`, `timerRunning = false`. After 1000ms timeout, set `shaking = false`. The state-update that follows will advance the turn.
    - On `state-update` received: if the `currentPlayer` changed (compare old and new snapshot), reset local display timer to the stored mode value.
