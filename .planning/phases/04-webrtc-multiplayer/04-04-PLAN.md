       - `let joinStatus = $state<'ready' | 'connecting' | 'error'>('ready')` -- for JoinOverlay
       - `let joinError = $state<string | null>(null)` -- error message for guest

       ```typescript
       const activeGameState = $derived(
         view === 'local-game' ? gameState :
         (view === 'online-host' || view === 'online-guest') ? onlineGameState :
         null
       );
       ```
       Use `activeGameState` for all HexCanvas/TurnIndicator/MoveCounter/GameOverlay props.

    4. **Online Game button handler** (from LandingPage `onOnlineGame`):
       - Create network state: `networkState = createNetworkState()`
       - Create online game state: `onlineGameState = createOnlineGameState('host', null, networkState)`
       - Set `view = 'online-host'`
       - Set `window.location.hash = onlineGameState.shareLink?.split('#')[1] ?? ''` (only after peer opens, so do this reactively or let the online-game-state handle it)

    5. **Guest flow** (when hash detected, `view === 'online-guest'`):
       - Show JoinOverlay with status `joinStatus`
       - On "Join Game" click:
         - Set `joinStatus = 'connecting'`
         - Create network state: `networkState = createNetworkState()`
         - Create online game state: `onlineGameState = createOnlineGameState('guest', gameId, networkState)`
         - Watch `networkState.status` reactively: when it becomes 'connected', hide JoinOverlay. When error occurs, set `joinStatus = 'error'` and `joinError = networkState.error`.
       - On "Back" click: set `view = 'landing'`, `gameId = null`, `window.location.hash = ''`, destroy online game state if exists.

    6. **Waiting overlay for host** (`view === 'online-host'`):
       - Show `<WaitingOverlay>` when `onlineGameState?.waitingForGuest` is true
       - `link` prop: `onlineGameState?.shareLink ?? ''`
       - `status` prop: `networkState?.status === 'disconnected' ? 'registering' : 'waiting'` (registering until PeerJS open event)
       - Hide WaitingOverlay when networkState.status becomes 'connected'

    7. **ConnectionStatus** for online views:
       - Render `<ConnectionStatus status={networkState.status} />` when `view === 'online-host' || view === 'online-guest'` AND the overlay is dismissed (game is active)
       - Per UI-SPEC: only visible during online games, hidden in local mode and on landing page

    8. **Rematch in online mode**: The GameOverlay `onRematch` callback already calls `activeGameState.rematch()`. Since `onlineGameState.rematch()` handles the P2P rematch protocol (sending rematch-request/accept), no additional wiring is needed.

    9. **Cleanup**: When navigating back to landing (if ever), call `onlineGameState?.destroy()` to clean up PeerJS connections.

    10. **Hash update for host**: After online game state is created and peer opens, update `window.location.hash` to the game ID. Do this in an `$effect` that watches `onlineGameState?.shareLink`.

    Conditional rendering template structure:
    ```svelte
    {#if view === 'landing'}
      <LandingPage onLocalGame={startLocalGame} onOnlineGame={startOnlineGame} />
      <ThemeToggle ... />
    {:else}
      <div class="game-container">
        <ThemeToggle ... />
        {#if activeGameState}
          <HexCanvas bind:debugActive gameState={activeGameState} {themeColors} />
          <TurnIndicator ... />
          <MoveCounter ... />
          {#if activeGameState.status === 'won' && activeGameState.winner}
            <GameOverlay winner={activeGameState.winner} onRematch={() => activeGameState.rematch()} />
          {/if}
        {/if}
        {#if view === 'online-host' && onlineGameState?.waitingForGuest}
          <WaitingOverlay link={onlineGameState.shareLink ?? ''} status={...} />
        {/if}
        {#if view === 'online-guest' && joinStatus !== 'connected_and_playing'}
          <JoinOverlay onJoin={handleJoin} status={joinStatus} error={joinError} onBack={handleBack} />
        {/if}
        {#if (view === 'online-host' || view === 'online-guest') && networkState}
          <ConnectionStatus status={networkState.status} />
        {/if}
        <DebugOverlay active={debugActive} />
      </div>
    {/if}
    ```

    Important: The JoinOverlay should be hidden once connection is established. Track this via `networkState.status === 'connected'` condition.
