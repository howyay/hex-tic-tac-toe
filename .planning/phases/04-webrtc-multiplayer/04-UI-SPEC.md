# Phase 4 — UI Design Contract

> Visual and interaction contract for WebRTC multiplayer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none -- hand-rolled Svelte components + Canvas 2D (inherited) |
| Icon library | none -- copy icon rendered as inline SVG |
| Font | system-ui, -apple-system, sans-serif (inherited from Phase 1/2/3) |

**Note:** Phase 4 introduces new DOM views (landing page, waiting overlay, join overlay, connection status) using the same hand-rolled Svelte + CSS custom property theming established in Phase 3. No external design system tool is needed.

---

## Spacing Scale

Inherited from Phase 3. Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Connection status dot size; inline padding |
| sm | 8px | Connection status gap between dot and text; button internal padding vertical |
| md | 16px | Landing page paragraph spacing; overlay inner padding |
| lg | 24px | Landing page gap between sections; button horizontal padding |
| xl | 32px | Landing page content horizontal padding |
| 2xl | 48px | Landing page vertical padding from viewport center |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:

- Touch target minimum: 44px -- all buttons (Local Game, Online Game, Join Game, Copy, Rematch) must meet this height minimum for mobile accessibility.

## Typography

Inherited from Phase 2/3. Phase 4 adds one new role (Display) for the landing page title.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Rules summary text, connection status text, shareable link text |
| Label | 20px | 600 (semibold) | 1.2 | Turn indicator (inherited), waiting/join overlay heading text |
| Heading | 28px | 600 (semibold) | 1.2 | Game over winner text (inherited) |
| Display | 36px | 600 (semibold) | 1.1 | Landing page game title "Hex Connect6" |

Four sizes, two weights. Display size is new for Phase 4 landing page only.

Font stack: `system-ui, -apple-system, sans-serif` (matches existing `app.css :root`).

## Color

All colors inherited from Phase 3 theme system. Phase 4 introduces three new semantic colors for connection status. These must be added to both `[data-theme="dark"]` and `[data-theme="light"]` CSS custom property blocks.

### New CSS Custom Properties

| Role | CSS Variable | Dark Value | Light Value | Usage |
|------|-------------|------------|-------------|-------|
| Connected | `--color-status-connected` | `#22c55e` | `#16a34a` | Connection status dot (green) |
| Connecting | `--color-status-connecting` | `#eab308` | `#ca8a04` | Connection status dot (yellow) |
| Disconnected | `--color-status-disconnected` | `#ef4444` | `#dc2626` | Connection status dot (red) |
| Link text | `--color-link-text` | `rgba(255, 255, 255, 0.8)` | `rgba(0, 0, 0, 0.7)` | Shareable link URL display |
| Copy success | `--color-copy-success` | `#22c55e` | `#16a34a` | "Copied!" feedback text (same as connected green) |

### Inherited Colors (from Phase 3, unchanged)

All existing CSS custom properties remain. The 60/30/10 split is unchanged:

| Proportion | Application in Phase 4 |
|------------|----------------------|
| 60% dominant | Background on landing page, game board, overlays |
| 30% secondary | Grid lines, overlay backdrops, rules text, connection status text |
| 10% accent | Player X/O colors, CTA buttons (Local Game, Online Game, Join Game, Copy) |

### Accent Reserved For (extended from Phase 3)

- Player X marks, hover preview, and win glow (canvas) -- inherited
- Player O marks, hover preview, and win glow (canvas) -- inherited
- Rematch button background (DOM) -- inherited
- Turn indicator player letter color coding -- inherited
- Theme toggle icon fill on hover -- inherited
- **NEW:** "Online Game" button background on landing page
- **NEW:** "Copy" button background on waiting overlay
- **NEW:** "Join Game" button background on join overlay
- Connection status dots use semantic colors (green/yellow/red), NOT accent colors

### Landing Page Background Hex Grid Preview (D-01)

The landing page shows a decorative hex grid in the background. This reuses the canvas renderer at a fixed zoom with no interaction:

| Property | Value |
|----------|-------|
| Grid stroke | Same as game: `var(--color-grid)` |
| Opacity | 0.5 (additional layer of transparency on top of grid stroke) |
| Animation | None -- static decorative background |
| Interaction | None -- pointer-events: none on the canvas |

## Interaction Contract

### Landing Page (D-01, D-02, D-03)

| Property | Value |
|----------|-------|
| Layout | Centered column: title, tagline, rules, two buttons stacked vertically |
| Content width | `max-width: 400px`, horizontally centered |
| Content position | Vertically centered with `display: flex; align-items: center; justify-content: center; min-height: 100%` |
| Background | Decorative hex grid canvas behind content (pointer-events: none, opacity: 0.5) |
| Theme toggle | Visible at `top: 8px; right: 8px` (same position as in-game, D-02) |
| Button layout | Two buttons stacked vertically, 16px gap between them |
| Button style | Same as rematch button: `var(--color-button-bg)` background, `var(--color-button-text)` text, 4px border-radius, `padding: 12px 24px`, 16px font-size, 600 weight |
| Button width | `width: 100%` within the content column (both buttons same width) |
| Button min-height | 44px (touch target) |
| Button hover | `var(--color-button-hover)` background |

### Guest Link Bypass (D-03, D-07)

| Property | Value |
|----------|-------|
| Trigger | Page loads with a hash fragment of 6+ characters (e.g. `#a1b2c3d4`) |
| Behavior | Skip landing page entirely. Show game board with JoinOverlay on top. |
| JoinOverlay | Centered overlay with "Join Game" heading and "Join Game" button |
| On Join click | Connect to host via PeerJS. Transition JoinOverlay to "Connecting..." state. |
| On connection success | Remove JoinOverlay. Game starts. ConnectionStatus appears in top-left. |
| On connection failure | Show error message in the overlay: "Game not found. The host may have left." with a "Back" link to landing page. |

### Host Flow (D-06, D-08)

| Property | Value |
|----------|-------|
| Trigger | User clicks "Online Game" on landing page |
| Step 1 | Transition to game board view. Show WaitingOverlay over the hex board. |
| Step 2 (PeerJS registering) | WaitingOverlay shows "Connecting..." while Peer registers with PeerJS Cloud. |
| Step 3 (Peer open) | WaitingOverlay shows "Waiting for opponent..." with the shareable link + Copy button. URL hash is set to the game ID. |
| Step 4 (Guest connects) | WaitingOverlay removed. Game starts. ConnectionStatus appears in top-left. Host sends `game-start` message. |
| Cancel | No explicit cancel button in v1. Host can navigate away or refresh. |

### Waiting Overlay

| Property | Value |
|----------|-------|
| Position | `position: absolute; inset: 0` (covers game board, same as GameOverlay) |
| Z-index | 20 (same layer as GameOverlay -- they are mutually exclusive) |
| Background | `var(--color-gameover-bg)` (reuses game over backdrop -- semi-transparent) |
| Layout | Flexbox column, centered both axes |
| Heading | "Waiting for opponent..." at 20px semibold, `var(--color-text)` |
| Link display | Full shareable URL in a styled container: `var(--color-overlay-bg)` background, `border-radius: 8px`, `padding: 12px 16px`, `font-family: monospace`, 16px regular, `var(--color-link-text)` color, `user-select: all` |
| Copy button | Below link container, 8px gap. Same button style as landing page buttons. Label: "Copy Link". |
| Copy feedback | On click, button text changes to "Copied!" in `var(--color-copy-success)` color for 2000ms, then reverts. |
| Spacing | 16px between heading and link container; 8px between link container and copy button |

### Join Overlay

| Property | Value |
|----------|-------|
| Position | `position: absolute; inset: 0` (covers game board) |
| Z-index | 20 |
| Background | `var(--color-gameover-bg)` |
| Layout | Flexbox column, centered both axes |
| Heading | "Join Game" at 20px semibold, `var(--color-text)` |
| Button | "Join Game" button, same style as landing page buttons. 16px gap below heading. |
| Connecting state | Button text changes to "Connecting..." and button becomes `pointer-events: none; opacity: 0.6` |
| Error state | Button replaced with error text (Body size, `var(--color-status-disconnected)`) and a "Back" text button (`var(--color-text-muted)`, underlined, navigates to landing page) |

### Connection Status Indicator (D-11, D-12)

| Property | Value |
|----------|-------|
| Position | `position: absolute; top: 12px; left: 12px` (D-11: top-left, opposite theme toggle) |
| Z-index | 10 (same layer as TurnIndicator, MoveCounter) |
| Layout | Inline flex row: dot + text |
| Dot | `width: 8px; height: 8px; border-radius: 50%` (circle) |
| Dot color | `var(--color-status-connected)` / `var(--color-status-connecting)` / `var(--color-status-disconnected)` |
| Connecting dot | Pulsing animation: `opacity: 1 -> 0.4 -> 1` over 1.5s, infinite, ease-in-out |
| Text | 14px regular (slightly smaller than Body to be unobtrusive), `var(--color-text-muted)` |
| Gap | 8px between dot and text |
| Visibility | Only visible during online games (hidden in local mode and on landing page) |
| pointer-events | none (informational only, not interactive) |
| user-select | none |

### View Routing

| View | Condition | Components Visible |
|------|-----------|-------------------|
| `landing` | No hash, initial load | LandingPage, ThemeToggle |
| `local-game` | User clicked "Local Game" | HexCanvas, TurnIndicator, MoveCounter, GameOverlay (when won), ThemeToggle, DebugOverlay |
| `online-host` | User clicked "Online Game" | HexCanvas, WaitingOverlay (until guest joins), TurnIndicator, MoveCounter, GameOverlay (when won), ConnectionStatus, ThemeToggle |
| `online-guest` | Hash detected on load | HexCanvas, JoinOverlay (until connected), TurnIndicator, MoveCounter, GameOverlay (when won), ConnectionStatus, ThemeToggle |

### Rematch in Online Mode (D-10)

| Property | Value |
|----------|-------|
| Trigger | Either player clicks "Rematch" in GameOverlay |
| Online behavior | Requesting player sends `rematch-request` message. Other player sees "Rematch" button as normal. When both agree, host computes new state via `applyRematch()` and sends `rematch-accept` with new snapshot. |
| Visual | Same GameOverlay as local mode. No additional UI for rematch negotiation -- the existing button works for both sides. If one player requests and the other hasn't yet, the requesting player's overlay remains visible. |
| Role persistence | Host stays X, guest stays O (D-10). Loser goes first per Phase 2 logic. |

### Mouse and Touch Interactions (inherited)

All existing interactions from Phase 1/2/3 remain unchanged:
- Click/tap to place stone
- Drag to pan
- Scroll/pinch to zoom
- Hover preview (desktop)
- Touch gesture discrimination (tap vs drag vs pinch)

Online mode adds one constraint: **guest input is disabled while waiting for host confirmation** of a move request. The canvas shows `cursor: wait` during this brief period (<100ms typical).

## Canvas Rendering Contract (unchanged from Phase 3)

No changes to the canvas rendering pipeline. All 10 render steps remain identical. The canvas renders the same game state regardless of whether the state comes from local or network play, because `online-game-state` conforms to the same `GameStateAPI` interface.

### Performance Budget (extended)

| Metric | Target |
|--------|--------|
| Landing page decorative canvas | 30fps acceptable (non-interactive, reduced priority) |
| Network state update to canvas redraw | < 1 frame (16ms) -- deserialize snapshot + update $state + trigger redraw |
| Copy to clipboard | < 50ms (async clipboard API) |
