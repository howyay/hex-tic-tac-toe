    **Create `src/lib/state/online-game-state.svelte.ts`:**

    This is the critical module. It wraps game logic with P2P networking while conforming to `GameStateAPI`.

    1. Export `createOnlineGameState(role: PlayerRole, gameId: string | null, networkState: NetworkStateAPI)` factory:
       - Internal state: `let snapshot = $state(createInitialSnapshot('X'))` (per D-09: host=X, guest=O)
       - Internal: `let rejectedHex = $state<HexCoord | null>(null)`, rejection timeout, grid state via `createGridState()`
       - Internal: `let conn: { send, destroy } | null = null` (the PeerJS connection handle, NOT in $state per anti-pattern in research)
       - Internal: `let waitingForConfirmation = $state(false)` (guest blocks input while awaiting host response per research Open Question 2)

    2. **Host role** (`role === 'host'`):
       - Call `createHost(callbacks)` from connection.ts
       - `onOpen(id)`: set `networkState.status = 'connecting'` (waiting for guest), set `networkState.gameId = id`
       - `onConnect()`: set `networkState.status = 'connected'`, send `{ type: 'game-start', snapshot: serializeSnapshot(snapshot) }` to guest
       - `onData(msg)`: handle messages:
         - `move-request`: validate via `isValidMove(snapshot, { q: msg.q, r: msg.r })`. If valid, apply via `applyMove` and send `{ type: 'state-update', snapshot: serializeSnapshot(snapshot) }`. If invalid, send current state as correction (per research Pattern 3). Update grid.needsRedraw.
         - `rematch-request`: compute new snapshot via `applyRematch(snapshot)`, send `{ type: 'rematch-accept', snapshot: serializeSnapshot(newSnapshot) }`, update local snapshot. Reset camera to DEFAULT_CAMERA. grid.needsRedraw.
         - `ping`: send `{ type: 'pong' }`
       - `onClose()`: set `networkState.status = 'disconnected'`
       - `onError(err)`: set `networkState.error = err.message`, `networkState.status = 'disconnected'`
       - `placeStone(hex)`: Host applies locally (authoritative). If `isValidMove`, apply `applyMove`, send `state-update` to guest. If invalid and hex occupied, show rejection flash (same logic as local game-state). grid.needsRedraw.

    3. **Guest role** (`role === 'guest'`):
       - Call `joinGame(gameId!, callbacks)` from connection.ts
       - `onOpen(id)`: set `networkState.status = 'connecting'`
       - `onConnect()`: set `networkState.status = 'connected'`
       - `onData(msg)`: handle messages:
         - `game-start`: set `snapshot = deserializeSnapshot(msg.snapshot)`. grid.needsRedraw.
         - `state-update`: set `snapshot = deserializeSnapshot(msg.snapshot)`. Set `waitingForConfirmation = false`. grid.needsRedraw.
         - `rematch-accept`: set `snapshot = deserializeSnapshot(msg.snapshot)`. Reset camera. grid.needsRedraw.
         - `pong`: no-op (keepalive)
       - `onClose()`: set `networkState.status = 'disconnected'`
       - `onError(err)`: set `networkState.error = err.message`, `networkState.status = 'disconnected'`
       - `placeStone(hex)`: Guest does NOT apply locally. Send `{ type: 'move-request', q: hex.q, r: hex.r }`. Set `waitingForConfirmation = true`. If hex is occupied locally, show rejection flash (client-side check to avoid unnecessary network round-trip).

    4. **rematch()**: Either role sends `{ type: 'rematch-request' }`. Host handles it by computing + broadcasting new state. Guest just sends the request and waits for `rematch-accept`.

    5. **Return object** must match `GameStateAPI` shape exactly:
       ```typescript
       return {
         get currentPlayer() { return derived currentPlayer },
         get board() { return derived board },
         get totalMoves() { return derived totalMoves },
         get placementsThisTurn() { return derived placementsThisTurn },
         get maxPlacements() { return derived maxPlacements },
         get status() { return derived status },
         get winner() { return derived winner },
         get winningLine() { return derived winningLine },
         get rejectedHex() { return rejectedHex },
         get gridState() { return grid },
         placeStone,
         rematch,
         // Additional network-specific getters:
         get networkStatus() { return networkState.status },
         get shareLink() { return gameId ? buildShareLink(gameId) : null },
         get waitingForGuest() { return role === 'host' && networkState.status === 'connecting' },
         get playerRole() { return role },
         destroy: () => conn?.destroy(),
       };
       ```

    6. Also export the return type: `export type OnlineGameStateAPI = ReturnType<typeof createOnlineGameState>`

    Important implementation notes:
    - Import `DEFAULT_CAMERA` from `src/lib/render/camera.ts` for rematch camera reset
    - Import `createGridState` from `src/lib/state/grid-state.svelte.ts` (same as local game state)
    - The $derived values must be computed from the reactive `snapshot` $state
    - Do NOT store the PeerJS Peer object in $state (per research anti-pattern)
    - Guest `placeStone` should be a no-op if `waitingForConfirmation` is true (prevents double-click per research Open Question 2)
    - Guest `placeStone` should also be a no-op if `snapshot.currentPlayer` doesn't match the guest's role ('O') -- guest can only place during their turn
    - Host `placeStone` should be a no-op if `snapshot.currentPlayer` doesn't match 'X' on host's turn -- host can only place during their turn
