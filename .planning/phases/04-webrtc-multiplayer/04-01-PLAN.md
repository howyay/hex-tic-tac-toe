       - `{ type: 'move-request'; q: number; r: number }` (guest to host)
       - `{ type: 'state-update'; snapshot: SerializedSnapshot }` (host to guest)
       - `{ type: 'game-start'; snapshot: SerializedSnapshot }` (host to guest)
       - `{ type: 'rematch-request' }` (either direction)

    5. Export all types and functions.

    3. Export `generateGameId` for testing.
    4. Export `buildShareLink(gameId: string): string` that returns `${window.location.origin}${window.location.pathname}#${gameId}`.
    5. Export `parseGameIdFromHash(): string | null` that reads `window.location.hash.slice(1)`, returns it if length >= 6 (per D-03), else null.
    6. Define `ConnectionCallbacks` interface:
       ```typescript
       export interface ConnectionCallbacks {
         onOpen: (gameId: string) => void;
         onConnect: () => void;
         onData: (msg: GameMessage) => void;
         onClose: () => void;
         onError: (err: Error) => void;
       }
       ```
    7. Export `createHost(callbacks: ConnectionCallbacks)` function:
       - Generates game ID via `generateGameId()`
       - Creates `new Peer(gameId, { debug: 0 })`
       - Wires `peer.on('open')` to `callbacks.onOpen(gameId)`
       - Wires `peer.on('connection')` to store the DataConnection and wire its events (open -> callbacks.onConnect, data -> callbacks.onData, close -> callbacks.onClose)
       - Wires `peer.on('error')` to callbacks.onError
       - Handles 'unavailable-id' error per research Open Question 1: regenerate ID and retry (max 3 attempts)
       - Returns `{ peer, send: (msg: GameMessage) => conn.send(msg), destroy: () => peer.destroy(), gameId }` (send is null-safe -- no-op if no connection yet)
    8. Export `joinGame(gameId: string, callbacks: ConnectionCallbacks)` function:
       - Creates `new Peer(undefined, { debug: 0 })` (random guest ID)
       - On `peer.on('open')`, connects via `peer.connect(gameId, { reliable: true })`
       - Wires DataConnection events: open -> callbacks.onConnect, data -> callbacks.onData, close -> callbacks.onClose
       - Handles `peer-unavailable` error per Pitfall 5 -- surfaces via callbacks.onError
       - Returns `{ peer, send: (msg: GameMessage) => conn.send(msg), destroy: () => peer.destroy() }`
