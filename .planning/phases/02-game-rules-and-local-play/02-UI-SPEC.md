| Property | Value |

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Turn indicator | "{X/O} -- {n} of {max}" | D-12. Examples: "X -- 1 of 1" (first turn), "O -- 1 of 2", "O -- 2 of 2" |
| Move counter | "Move {n}" | D-13. Increments on each stone placed. Example: "Move 7" |
| Game over winner | "X wins!" or "O wins!" | D-08. Uses X/O identity, not color names |
| Rematch button | "Rematch" | D-09. Single button label |
| Rejection feedback | (no text -- visual flash only) | D-02. Brief red flash on occupied hex, no toast/text |
| Empty state | not applicable | Game board is always visible; game starts immediately |
| Error state | not applicable | No error conditions in local hot-seat play |
| Destructive confirmation | not applicable | Rematch is not destructive -- it starts a fresh game |

## Interaction Contract

### Stone Placement (GAME-01, GAME-02, D-01)

| Property | Value |
|----------|-------|
| Trigger | mouseup after click (not drag) on an empty hex |
| Click vs drag threshold | 5px cumulative mouse movement between mousedown and mouseup |
| Behavior | Stone appears instantly at clicked hex in current player color |
| Invalid placement | Brief red flash (150ms) on occupied hex, then clear. No stone placed. |
| Turn flow | First turn: X places 1 stone. All subsequent turns: current player places 2 stones. Counter updates between placements. |
| Win check | After EVERY stone placement (not just end of turn). Game ends immediately if 6-in-a-row found. |
| Cursor | `pointer` on empty hex during play, `not-allowed` on occupied hex |

### Hover Preview (updated from Phase 1)

| Property | Value |
|----------|-------|
| Trigger | mousemove (no button pressed), game status = playing |
| Behavior | Hex under cursor filled with CURRENT player color at 0.3 opacity (was fixed blue in Phase 1) |
| Player X hover | `rgba(79, 195, 247, 0.3)` |
| Player O hover | `rgba(239, 83, 80, 0.3)` |
| Occupied hex hover | No preview shown (hex already has a mark) |
| Game over | No hover preview when status != playing |

### Win Presentation (GAME-06, D-05, D-06)

| Property | Value |
|----------|-------|
| Win highlight | 6 winning hexes get a glowing outline: `shadowBlur: 15`, `shadowColor: player color`, `strokeStyle: player color`, `lineWidth: 3` |
| Board freeze | Pan and zoom disabled when game status = won (D-06) |
| Auto-center | Animate camera to center on the bounding box of the winning 6 hexes over 300ms ease-out (discretionary) |
| Game over overlay | Appears after 500ms delay (let player see the winning line first) |

### Game Over Overlay (GAME-07, D-07, D-08, D-09)

| Property | Value |
|----------|-------|
| Layout | Full-viewport semi-transparent backdrop with centered content |
| Backdrop | `rgba(0, 0, 0, 0.7)` covering entire canvas |
| Winner text | "{X/O} wins!" at 28px semibold, white, centered |
| Rematch button | "Rematch" at 16px semibold, accent background, dark text, 8px vertical / 24px horizontal padding, 4px border-radius |
| Button hover | Background lightens to `#81d4fa` |
| Spacing | 24px gap between winner text and rematch button |
| Animation | Fade in over 200ms (opacity 0 to 1) |

### Rematch (GAME-08, GAME-09, D-10)

| Property | Value |
|----------|-------|
| Trigger | Click "Rematch" button |
| Behavior | All game state resets. Board clears. Game over overlay removed. Pan/zoom re-enabled. Camera resets to origin. |
| First player swap | Loser of previous game goes first (D-10) |
| Transition | Instant -- no animation on reset |

### Rejection Animation (D-02)

| Property | Value |
|----------|-------|
| Trigger | Click on an occupied hex while game status = playing |
| Animation | Red flash fill (`rgba(255, 80, 80, 0.4)`) on the occupied hex for 150ms, then clear |
| Implementation | Set a "rejected hex" in state with timestamp. Renderer draws flash fill. Clear after 150ms via setTimeout. |

### Pan and Zoom (inherited from Phase 1, modified)

| Property | Value |
|----------|-------|
| During play | Pan and zoom work normally (same as Phase 1) |
| During game over | Pan and zoom DISABLED (D-06). Cursor reverts to default. |
| After rematch | Pan and zoom re-enabled |

## Canvas Rendering Contract (updated from Phase 1)

### Render Order (back to front)

1. Clear canvas with background `#1a1a2e`
2. Apply camera transform via `ctx.setTransform()`
3. Draw hex grid outlines for visible hexes (viewport-culled)
4. **Draw placed stones** -- iterate board Map, draw X or O mark at each hex center
5. **Draw win highlight** -- if game status = won, draw glowing outline on winning 6 hexes
6. **Draw rejection flash** -- if a rejected hex is active, draw red flash fill
7. Draw hover preview on hovered hex (only if game is playing and hex is empty)
8. Reset transform to screen space
9. Draw edge fade overlay (4 gradients)
10. Draw debug coordinate overlay (if enabled)

### Stone Mark Rendering

| Mark | Drawing Method | Color | Dimensions |
|------|---------------|-------|------------|
| X | Two crossed lines (canvas lineTo) | `#4fc3f7` | radius = hexSize * 0.55, lineWidth = 3, lineCap = round |
| O | Circle (canvas arc) | `#ef5350` | radius = hexSize * 0.55, lineWidth = 3, lineCap = round |

### Performance Budget (inherited, extended)

| Metric | Target |
|--------|--------|
| Frame rate | 60fps during play, pan, zoom |
| Max stones on board | ~200 (typical game ends well before this) |
| Stone draw cost | < 0.02ms per mark (2 lines or 1 arc) |
| Win glow cost | < 0.5ms total (6 hexes with shadowBlur, only during game over) |

## DOM Overlay Layout

All DOM overlays are positioned absolutely within the game container (sibling to canvas).

### Turn Indicator (UI-01, UI-02, D-11, D-12)

| Property | Value |
|----------|-------|
| Position | `position: absolute; top: 16px; left: 50%; transform: translateX(-50%)` |
| Background | `rgba(0, 0, 0, 0.5)` with `border-radius: 20px` (pill shape) |
| Padding | 8px vertical, 16px horizontal |
| Text | 20px semibold, white. Player letter colored: X in `#4fc3f7`, O in `#ef5350` |
| Format | "{colored letter} -- {n} of {max}" |
| Visibility | Always visible during gameplay; hidden during game over overlay |
| Z-index | 10 (above canvas, below game over overlay) |

### Move Counter (UI-03, D-13)

| Property | Value |
|----------|-------|
| Position | `position: absolute; bottom: 16px; right: 16px` |
| Text | 16px regular, `rgba(255, 255, 255, 0.6)` (muted) |
| Format | "Move {n}" |
| Background | none (text only, subtle) |
| Z-index | 10 |

### Game Over Overlay (GAME-07, D-07)

| Property | Value |
|----------|-------|
| Position | `position: absolute; inset: 0` (covers entire container) |
| Z-index | 20 (above all other overlays) |
| Background | `rgba(0, 0, 0, 0.7)` |
| Content layout | Flexbox, column, centered both axes |
| Winner text | 28px semibold, `#ffffff` |
| Rematch button | 16px semibold, `#4fc3f7` background, `#1a1a2e` text, `padding: 8px 24px`, `border-radius: 4px`, `border: none`, `cursor: pointer` |
| Entry animation | `opacity: 0 -> 1` over 200ms, delayed 500ms after win detection |
