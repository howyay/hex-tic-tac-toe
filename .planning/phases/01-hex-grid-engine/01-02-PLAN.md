  <action>
    **Create `src/lib/render/renderer.ts`:**

    Implements the canvas drawing pipeline. Key functions:

    `applyCamera(ctx, camera)`: Apply camera transform via `ctx.setTransform(camera.zoom, 0, 0, camera.zoom, camera.x * camera.zoom, camera.y * camera.zoom)`.

    `drawHex(ctx, center, size, strokeColor, fillColor?)`: Draw a single hex outline using `hexCorners()`. Use `ctx.beginPath()`, `moveTo`, `lineTo` for 6 corners, `closePath()`, optional fill, then stroke. `lineWidth = 1`.

    `drawGrid(ctx, hexes, hexSize, camera, zoom)`:
    - For each hex in the `hexes` array, compute pixel center via `hexToPixel(hex, hexSize)`
    - If `zoom >= 0.4`: draw full hex outline with stroke color `rgba(255, 255, 255, 0.12)` per UI-SPEC (D-02)
    - If `zoom < 0.4`: draw LOD dot instead (3px filled circle, color `rgba(255, 255, 255, 0.2)`) per D-07

    `render(ctx, camera, canvasWidth, canvasHeight, hexSize, hoveredHex, debugCoords)`:
    Full render pipeline per UI-SPEC render order:
    1. `ctx.setTransform(1,0,0,1,0,0)` then `ctx.clearRect(0,0,canvasWidth,canvasHeight)` then fill with `#1a1a2e`
    2. Apply camera transform via `applyCamera(ctx, camera)`
    3. Get visible hexes via `getVisibleHexes(camera, canvasWidth, canvasHeight, hexSize)`
    4. Call `drawGrid(ctx, hexes, hexSize, camera, camera.zoom)`
    5. Draw hover preview if `hoveredHex` is not null: fill that hex with current player color at 0.3 opacity (use `#4fc3f7` for X as default for Phase 1, alpha 0.3 = `rgba(79, 195, 247, 0.3)`)
    6. Reset transform: `ctx.setTransform(1,0,0,1,0,0)`
    7. Call `drawEdgeFade(ctx, canvasWidth, canvasHeight, 64)` from effects module
    8. If debugCoords enabled, draw coordinate labels at each visible hex center (transform back to screen space for each, draw `"(q,r)"` text in `rgba(255, 255, 255, 0.5)` font `10px system-ui`)

    **Create `src/lib/render/effects.ts`:**

    `drawEdgeFade(ctx, width, height, fadeSize)`:
    Draw 4 linear gradients in screen space (transform already reset before calling).
    - Top: gradient from (0,0) to (0, fadeSize), `rgba(26, 26, 46, 1.0)` to `rgba(26, 26, 46, 0.0)`
    - Bottom: gradient from (0, height) to (0, height - fadeSize), same colors
    - Left: gradient from (0,0) to (fadeSize, 0), same colors
    - Right: gradient from (width, 0) to (width - fadeSize, 0), same colors
    Each gradient fills its respective edge strip.

    `drawLODDot(ctx, center, radius, color)`:
    Draw a filled circle at `center` with given `radius` and `color`. Used when `zoom < 0.4` per D-07.

    **Create `src/components/HexCanvas.svelte`:**

    This is the main interactive component. It must:

    1. **Canvas setup with HiDPI support:**
       - Bind a `<canvas>` element
       - On mount and window resize: set `canvas.width = canvas.clientWidth * devicePixelRatio`, `canvas.height = canvas.clientHeight * devicePixelRatio`, then `ctx.scale(dpr, dpr)`. Use `canvas.clientWidth` and `canvas.clientHeight` (CSS size) for all world-space calculations.
       - Canvas CSS: `width: 100%; height: 100%; display: block; cursor: grab;`

    2. **Reactive state:**
       - Call `createGridState()` from grid-state.svelte.ts to get state object
       - `const HEX_SIZE = 30;` (hex radius per UI-SPEC)

    3. **rAF render loop** (per RESEARCH.md Pattern 4):
       ```
       $effect(() => {
         let animationId: number;
         const loop = () => {
           if (state.needsRedraw) {
             const ctx = canvas.getContext('2d')!;
             const dpr = window.devicePixelRatio || 1;
             render(ctx, state.camera, canvas.clientWidth, canvas.clientHeight, HEX_SIZE, state.hoveredHex, state.debugCoords);
             state.needsRedraw = false;
           }
           animationId = requestAnimationFrame(loop);
         };
         animationId = requestAnimationFrame(loop);
         return () => cancelAnimationFrame(animationId);
       });
       ```

    4. **Pan handling (GRID-04, D-05):**
       - `onmousedown` (left button): set `isPanning = true`, record `lastMouseX/Y`
       - `onmousemove` while panning: compute delta `(e.clientX - lastX, e.clientY - lastY)`, call `panCamera(camera, deltaX, deltaY)`, update `state.camera`, update lastX/Y
       - `onmouseup` / `onmouseleave`: set `isPanning = false`
       - While panning: `cursor: grabbing`; idle: `cursor: grab`

    5. **Zoom handling (GRID-05, D-05):**
       - `onwheel` (with `{ passive: false }` and `e.preventDefault()` to stop page scroll):
       - Compute `zoomDelta = e.deltaY * 0.001`
       - Get cursor position relative to canvas: `{ x: e.offsetX, y: e.offsetY }`
       - Call `zoomAtPoint(state.camera, cursorPoint, zoomDelta)`
       - Update `state.camera`

    6. **Hover handling (GRID-06):**
       - `onmousemove` when NOT panning:
       - Convert mouse position to world space: `screenToWorld({ x: e.offsetX, y: e.offsetY }, state.camera)`
       - Convert world point to hex: `hexRound(...pixelToHex(worldPoint, HEX_SIZE))`
       - Set `state.hoveredHex = hex`
       - `onmouseleave`: set `state.hoveredHex = null`

    7. **Debug toggle (D-11):**
       - Listen for `keydown` on window: if `(e.ctrlKey || e.metaKey) && e.key === 'd'`, call `e.preventDefault()`, toggle `state.debugCoords`

    8. **Window resize handler:**
       - On window resize, re-scale canvas for DPI and set `state.needsRedraw = true`

    **Update `src/App.svelte`:**
    ```svelte

    ```

    **Important implementation details:**
    - Do NOT call `ctx.save()/ctx.restore()` per hex -- use `setTransform()` once per frame (anti-pattern from RESEARCH.md)
    - Do NOT draw inside `$effect` directly -- use rAF loop with dirty flag (RESEARCH.md anti-pattern)
    - Always reset transform before `clearRect`: `ctx.setTransform(1,0,0,1,0,0)` (RESEARCH.md pitfall)
    - The render function in renderer.ts handles DPI scaling via the ctx that already has `ctx.scale(dpr, dpr)` applied. Pass `canvas.clientWidth` and `canvas.clientHeight` (not canvas.width/height) to render.
