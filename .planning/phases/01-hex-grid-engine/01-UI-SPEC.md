# Phase 1 — UI Design Contract

> Visual and interaction contract for the hex grid engine phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — pure HTML5 Canvas rendering |
| Icon library | none — no icons in this phase |
| Font | system-ui, -apple-system, sans-serif (debug overlay only) |

**Note:** Phase 1 is entirely canvas-rendered. There are no DOM-based UI components, no buttons, no cards, no forms. The design contract below specifies canvas rendering tokens and interaction contracts rather than component-level tokens. DOM-level design tokens (component spacing, typography scale, color themes) will be established in Phase 2 when the first DOM UI elements appear.

### Canvas Rendering Geometry

> These are geometric rendering constants derived from hex math, not layout spacing tokens. They are exempt from the 4px grid constraint.

| Token | Value | Usage | Source |
|-------|-------|-------|--------|
| hex-size | 30px | Hex radius at default zoom (center to vertex) | RESEARCH.md discretionary recommendation |
| hex-width | 52px | Computed: sqrt(3) * 30 = ~52px (pointy-top) | Derived from hex-size |
| hex-height | 60px | Computed: 2 * 30 = 60px (pointy-top) | Derived from hex-size |
| hex-outline-width | 1px | Stroke width for hex cell outlines at default zoom | D-02: subtle outlines |
| edge-fade-size | 64px | Gradient fade band from each viewport edge | RESEARCH.md: 60-80px, rounded to spacing scale |
| viewport-cull-padding | 2 hexes | Extra hexes rendered beyond visible viewport | RESEARCH.md: prevent pop-in |

## Color

Phase 1 uses a dark-dominant palette optimized for a game board. All colors are specified as canvas rendering values.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) — background | `#1a1a2e` | Canvas clear color, edge fade target | Dark surface for game board |
| Secondary (30%) — grid | `rgba(255, 255, 255, 0.12)` | Hex cell outline stroke color | D-02: subtle outlines |
| Accent — Player X | `#4fc3f7` | X mark color, X hover preview | D-03: distinct player colors |
| Accent — Player O | `#ef5350` | O mark color, O hover preview | D-03: distinct player colors |
| Hover preview | Current player color at 0.3 opacity | Semi-transparent fill inside hovered hex | RESEARCH.md: 0.3 opacity |
| Debug text | `rgba(255, 255, 255, 0.5)` | Coordinate labels in debug mode | Low contrast to avoid distraction |
| LOD dot | `rgba(255, 255, 255, 0.2)` | Simplified hex dot at far zoom | D-07: simplified rendering |

### Edge Fade Gradient

The edge fade (D-04) uses the dominant background color transitioning from fully opaque at the viewport edge to fully transparent 64px inward:
- Outer edge: `rgba(26, 26, 46, 1.0)` (dominant background, opaque)
- Inner edge: `rgba(26, 26, 46, 0.0)` (dominant background, transparent)

Applied as four linear gradients (top, bottom, left, right) drawn in screen space after the grid.

### Accent Reserved For

- Player X stone marks and hover preview only
- Player O stone marks and hover preview only
- No other uses of accent colors in Phase 1

### Color Notes

- These colors establish the dark theme baseline. Phase 3 (UI-04) will introduce light/dark toggle. For Phase 1, dark theme only.
- Grid outline opacity (`0.12`) is deliberately low to achieve "subtle" per D-02. If hexes are invisible on screen, increase to `0.18` maximum.

## Interaction Contract

Since Phase 1 is a canvas interaction phase, the interaction contract replaces the typical component contract.

### Pan (GRID-04)

| Property | Value |
|----------|-------|
| Trigger | mousedown + mousemove (left button) |
| Behavior | Camera translates by mouse delta in world space |
| Cursor | `grab` on idle, `grabbing` during pan |
| Inertia | none (Phase 1 scope) |
| Boundary | none — infinite pan in all directions (D-08, GRID-02) |

### Zoom (GRID-05)

| Property | Value |
|----------|-------|
| Trigger | wheel event (deltaY) |
| Behavior | Zoom anchored to cursor position (D-05) |
| Zoom min | 0.25 (~50 hexes visible, per D-06) |
| Zoom max | 3.0 (~10 hexes visible, per D-06) |
| Zoom default | 1.0 |
| Zoom step | multiply by `1 - (deltaY * 0.001)`, clamped to min/max |
| LOD threshold | zoom < 0.4: switch to dot rendering (D-07) |

### Hover (GRID-06)

| Property | Value |
|----------|-------|
| Trigger | mousemove (no button pressed) |
| Behavior | Hex under cursor filled with current player color at 0.3 opacity |
| Transition | instant — no animation, updates every mousemove |
| Clear | hover preview removed when cursor leaves canvas |

### Debug Coordinate Toggle (D-11)

| Property | Value |
|----------|-------|
| Trigger | Ctrl+D (Windows/Linux), Cmd+D (macOS) |
| Behavior | Toggle coordinate labels at hex centers showing "(q, r)" |
| Default state | hidden |
| Persistence | session only (not persisted across page loads) |

---

## Level of Detail (LOD) Contract (D-07)

| Zoom Range | Rendering |
|------------|-----------|
| zoom >= 0.4 | Full hex outlines + X/O marks + hover preview + debug coords (if enabled) |
| zoom < 0.4 | Hex outlines hidden. Each hex center rendered as a 3px filled circle. X/O marks hidden. Hover preview still active (dot highlight). Debug coords hidden. |

## Canvas Rendering Contract

### Render Order (back to front)

1. Clear canvas with dominant background color (`#1a1a2e`)
2. Reset transform, apply camera transform via `ctx.setTransform()`
3. Draw hex grid outlines for all visible hexes (viewport-culled)
4. Draw placed pieces (X and O marks) — Phase 2 adds these, but renderer must support the layer
5. Draw hover preview on hovered hex
6. Reset transform to screen space
7. Draw edge fade overlay (4 gradients)
8. Draw debug coordinate overlay (if enabled)

### Performance Budget

| Metric | Target |
|--------|--------|
| Frame rate | 60fps during idle, pan, and zoom |
| Max hexes per frame | ~2500 (50x50 grid at max zoom-out) |
| Hex draw cost | < 0.01ms per hex (outline path + stroke) |
| Frame budget | < 16ms total render time |

### Retina / HiDPI Support

Canvas must scale for `devicePixelRatio`:
- `canvas.width = canvas.clientWidth * dpr`
- `canvas.height = canvas.clientHeight * dpr`
- `ctx.scale(dpr, dpr)`
- CSS: `width: 100%; height: 100%`
